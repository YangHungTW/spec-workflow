## 3. Risks

- **Migrate flag-set drift** — `bin/scaff-seed migrate`'s flag surface (`--from`, `--ref`) may differ from what the developer expects. Mitigation: PRD §Decisions pins the flag-set to `--from "$REPO_ROOT"` with implicit-HEAD ref, mirroring sibling chore `20260426-chore-t108-migrate-coverage` A5; the developer should grep `cmd_migrate` argv parsing (around `bin/scaff-seed:1300`) before invoking and confirm the recorded ref shape matches.
- **Manifest-shape brittleness** — running `scaff-seed init` first to author a manifest implies the test inherits any future change to manifest format. Mitigation: keep the seeding step minimal (just `scaff-seed init`); if a future feature changes manifest shape, this test self-heals because it rebuilds the manifest each run.
