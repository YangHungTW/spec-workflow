# 02-design — rename-flow-monitor

This folder contains the design deliverable for the `20260421-rename-flow-monitor` feature. Because this feature is a brand-rename follow-up (not a new screen or interaction), there is no visual mockup. The UI layout, visual design, and interaction model of flow-monitor are unchanged; only string literals that embed the old brand (`specflow`, `Specflow`, `.spec-workflow/`) are being updated to the new brand (`scaff`, `Scaff`, `.specaffold/`).

The authoritative deliverable is `string-delta.md`, which contains a byte-exact substitution table covering every user-visible string that changes: i18n JSON keys in `en.json` and `zh-TW.json`, in-code JSDoc comments in TSX components, path literals rendered to the user in `CardDetail.tsx` and `SettingsRepositories.tsx`, and user-facing paragraphs in `README.md`. The Developer should treat each row in the table as a precise find-and-replace instruction and must not deviate from the replacement values without reopening this design stage. QA stages should validate every replacement value against the `legacy` column (confirm it no longer appears in non-test source) and the `replacement` column (confirm it now appears and is spelled exactly as specified).
