Feature Request — Flow Monitor B1

A read-only macOS desktop monitoring tool that watches .spec-workflow/features/*/STATUS.md files across one or more repositories and displays session health at a glance.

Problem

When multiple specflow sessions are in-flight across repos it is easy to lose track of which session is stalled waiting for user input. There is no ambient indicator.

Success criteria (initial)

  • See all sessions in one panel without switching terminal tabs
  • Know at a glance which session is blocked and for how long
  • Receive a macOS notification when a session goes stalled

Brainstorm — Flow Monitor B1

Core concept

Poll STATUS.md files every N seconds; derive health state from last-modified timestamp relative to two configurable thresholds (stale > T1, stalled > T2).

UX candidates explored

  • Floating compact panel (always-on-top, minimal footprint)
  • Full main window with project switcher sidebar and session card grid
  • Tray icon only (rejected — too low information density)

Decision: ship both

Main window for deep inspection; compact panel for ambient monitoring. Toggle between them from the toolbar.

Notifications

macOS NSUserNotification / UNUserNotificationCenter — fires once per stalled transition, no repeat. Sound: off by default.

Design — 02-design/

This stage contains a folder, not a single file. Files in .spec-workflow/features/20260419-flow-monitor/02-design/:

  • mockup.html self-contained HTML · 7 screens
  • notes.md design decisions + open questions
  • README.md how to open + screens table

Acceptance Criteria (card detail)

  • AC7.a — Clicking any card in the main window navigates to a full-window detail view (not a modal)
  • AC7.b — Breadcrumb shows: ← All Sessions / [repo] / [slug]; back arrow returns to main
  • AC7.c — Left rail shows 11-stage checklist (display only, no toggle) and Notes timeline (latest first)
  • AC7.d — Right pane shows tab strip for all 8 doc slots; existing files are full-tone, future-stage files are muted with tooltip "Not yet generated"
  • AC7.e — Default tab = latest existing document
  • AC7.f — Markdown pane footer reads "Read-only preview. Open in Finder to edit."
  • AC7.g — Action row: "Open in Finder" and "Copy path" only (no edit, no advance-stage)

Out of scope — B2

  • Inline checkbox toggle on stage checklist
  • "Advance stage" / "Send instruction" button
  • Comment or annotation affordance
  • Card detail in compact panel (compact panel clicking opens main window detail)

Design notes — Card Detail screen

  • • Navigation pattern: full-window master-detail (not a modal); same window chrome as Main Window; back arrow returns to all-sessions grid.
  • • Left rail (260px fixed): 11-stage checklist (vertical list — natural for 11 items + timeline below) + Notes timeline latest-first below a divider.
  • • Right pane (flex-1): tab strip across top; active tab underline in ink-green (#1B4332 dark) / indigo (light); future-stage tabs muted+italic with "Not yet generated" tooltip.
  • • Default tab: latest existing document (03 prd in this example — brainstorm done, design done, prd done, tech not yet).
  • • 02 design tab: shows a file index (not parsed markdown) since the design stage is a folder, not a single file.
  • • Read-only reinforcement: footer "Read-only preview. Open in Finder to edit." anchors the B1 contract on every tab.
  • • Actions: "Open in Finder" + "Copy path" only — consistent with PRD R7 / AC7.g.
  • • Compact panel click: clicking a session in the compact panel opens the main window at this card-detail view (no compact-detail screen needed).
